## Creating New Templates

### Template Creation Process

Build templates with the visual designer:

1. **Choose Template Type**: Select document category
2. **Start with Base**: Use blank or existing template
3. **Add Elements**: Drag components to canvas
4. **Configure Properties**: Set element attributes
5. **Preview and Test**: Verify with sample data
6. **Save and Deploy**: Make template available

### Design Elements

#### Text Elements

- **Static Text**: Fixed labels and headers
- **Dynamic Fields**: Variable content from data
- **Rich Text**: Formatted text with styling
- **Multi-line Text**: Paragraphs and descriptions

#### Layout Components

- **Headers**: Document titles and branding
- **Footers**: Contact information and disclaimers
- **Tables**: Structured data display
- **Sections**: Grouped content areas
- **Dividers**: Visual content separation

#### Data Fields

- **Order Information**: Order numbers, dates, status
- **Customer Data**: Names, addresses, contact info
- **Product Details**: Items, quantities, prices
- **Shipping Info**: Carriers, services, tracking
- **Company Details**: Business information

> [!TIP]
>
> **Template Variables**
>
> Use template variables like `{order.order_id}` or `{customer.name}` to dynamically populate document content. The system automatically replaces these with actual data when generating documents.

## Template Variables and Data

### Available Data Sources

Templates can access various data:

#### Order Data

```handlebars
<!-- Order Information -->
{{order.order_id}}
{{order.created_at}}
{{order.status}}
{{order.total_value}}
{{order.currency}}

<!-- Line Items -->
{{#each order.line_items}}
  {{title}}
  - Qty:
  {{quantity}}
  - Price:
  {{value_amount}}
{{/each}}
```

#### Customer Information

```handlebars
<!-- Shipping Address -->
{{order.shipping_to.person_name}}
{{order.shipping_to.address_line1}}
{{order.shipping_to.city}},
{{order.shipping_to.state_code}}
{{order.shipping_to.postal_code}}

<!-- Contact Details -->
{{order.shipping_to.email}}
{{order.shipping_to.phone_number}}
```

#### Company Details

```handlebars
<!-- Business Information -->
{{company.name}}
{{company.address}}
{{company.phone}}
{{company.email}}
{{company.website}}
{{company.logo_url}}
```

#### Shipment Data

```handlebars
<!-- Shipping Information -->
{{shipment.tracking_number}}
{{shipment.carrier_name}}
{{shipment.service}}
{{shipment.estimated_delivery}}
{{shipment.shipping_cost}}
```

### Conditional Logic

Show content based on conditions:

```handlebars
<!-- Show only for international orders -->
{{#if order.is_international}}
  <div>Customs Declaration Required</div>
{{/if}}

<!-- Display different content by status -->
{{#switch order.status}}
  {{#case "fulfilled"}}
    <div>Order Complete</div>
  {{/case}}
  {{#case "partial"}}
    <div>Partially Fulfilled</div>
  {{/case}}
  {{#default}}
    <div>Processing Order</div>
  {{/default}}
{{/switch}}
```

### Calculations and Formatting

#### Numeric Formatting

```handlebars
<!-- Currency formatting -->
{{formatCurrency order.total_value order.currency}}

<!-- Number formatting -->
{{formatNumber item.quantity 0}}

<!-- Percentage -->
{{formatPercent discount_rate 2}}
```

#### Date Formatting

```handlebars
<!-- Date formats -->
{{formatDate order.created_at "MM/DD/YYYY"}}
{{formatDate shipment.estimated_delivery "MMMM Do, YYYY"}}
{{formatTime shipment.pickup_time "h:mm A"}}
```

> [!WARNING]
>
> **Data Availability**
>
> Template variables depend on the context where the template is used. Not all data is available in every situation. Test templates thoroughly with different data scenarios.

## Advanced Template Features

### Dynamic Tables

Create tables with variable row counts:

```handlebars
<!-- Order items table -->
<table>
  <thead>
    <tr>
      <th>Item</th>
      <th>Quantity</th>
      <th>Price</th>
      <th>Total</th>
    </tr>
  </thead>
  <tbody>
    {{#each order.line_items}}
    <tr>
      <td>{{title}}</td>
      <td>{{quantity}}</td>
      <td>{{formatCurrency value_amount ../order.currency}}</td>
      <td>{{formatCurrency (multiply quantity value_amount) ../order.currency}}</td>
    </tr>
    {{/each}}
  </tbody>
</table>
```

### Conditional Sections

Show sections based on data:

```handlebars
<!-- International shipping section -->
{{#if order.is_international}}
<div class="customs-section">
  <h3>Customs Information</h3>
  <p>Customs Value: {{formatCurrency order.declared_value order.currency}}</p>
  <p>Country of Origin: {{order.origin_country}}</p>

  {{#each order.customs_items}}
  <div>
    <span>{{description}}: {{quantity}} x {{formatCurrency unit_value ../order.currency}}</span>
  </div>
  {{/each}}
</div>
{{/if}}
```

## Troubleshooting

### Common Template Issues

#### Design Problems

- **Element Positioning**: Objects not aligning correctly
- **Data Missing**: Variables not displaying values
- **Formatting Issues**: Text or numbers displaying incorrectly
- **Layout Problems**: Elements overlapping or misplaced

#### Data Issues

- **Variable Errors**: Template variables not resolving
- **Missing Information**: Required data not available
- **Formatting Problems**: Dates, numbers, currency display issues
- **Conditional Logic**: If/then statements not working

#### Generation Failures

- **Template Errors**: Syntax or logic problems
- **Data Access**: Cannot retrieve order/shipment information
- **Output Problems**: PDF generation failures
- **Performance Issues**: Slow document generation

### Debugging Templates

#### Testing Approach

1. **Use Sample Data**: Test with known good data
2. **Check Variables**: Verify all variables resolve correctly
3. **Validate Logic**: Test conditional statements
4. **Review Output**: Check generated document quality
5. **Performance Testing**: Verify generation speed

#### Common Solutions

- **Variable Syntax**: Ensure correct Handlebars syntax
- **Data Availability**: Check data exists in context
- **CSS Conflicts**: Resolve styling issues
- **Template Structure**: Verify proper HTML structure

## API Integration

### Programmatic Template Management

Manage templates via API:

#### Template Operations

```javascript
// Get available templates
const templates = await karrio.templates.list({
  type: "invoice",
  active: true,
});

// Generate document
const document = await karrio.documents.generate({
  template_id: "invoice_template_123",
  order_id: "order_456",
  format: "pdf",
});

// Download generated document
const documentUrl = document.document_url;
```

#### Batch Generation

```javascript
// Generate multiple documents
const batchRequest = {
  template_id: "packing_slip_template",
  orders: ["order_1", "order_2", "order_3"],
  format: "pdf",
};

const batchResult = await karrio.documents.batchGenerate(batchRequest);
```
